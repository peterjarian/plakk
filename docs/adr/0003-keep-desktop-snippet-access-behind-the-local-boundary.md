# Keep desktop snippet access behind the local boundary

#48–#51 will establish a desktop boundary where renderer surfaces access snippet metadata, content, commands, and projected work only through IPC into Electron main's local replica, managed-content store, and durable outboxes. Electron main will alone synchronize authoritative snippet state with the backend; renderer access to authentication, account, billing, and storage-setup services may remain server-facing because those services do not own snippet state. Recording the target now prevents compatibility work in #47 from becoming the architecture by accident; the boundary costs additional local commands and projections but preserves offline behavior, background ownership, and one coherent source of desktop snippet state.
